- **批次**：第三批（media 讲义批量入库）
- **状态**：⬜ 待开始
- **来源**：[`PLAN-B-整章讲义入库.md`](../PLAN-B-整章讲义入库.md)
- **VLM 成本**：0（文字型，fitz 抽文字）

## What to build

一个**离线脚本**：从 3 份整章讲义抽取「考点 → 知识点 → 讲解正文」层级结构，与官方考纲种子 diff，**产出待人工评审清单**。本切片**不写入** S4 已锁定的知识点表（只读对比 + 产出清单）。

来源（3 份，`media/`，均文字型）：
- 852251 = 第三章 合伙企业法律制度
- 858684 = 第四章 物权法律制度
- 746175 = 第五章 合同法律制度

端到端：

1. fitz 抽全文，**剔除固定页眉页脚**（`第X章…|强化班-周周` / `学会计就到之了课堂`）。
2. 识别层级：考点标题、知识点小标题、讲解正文。纯文本丢层级时，用 fitz 的 span **字号/缩进**信息辅助判层。
3. 与 [`data/syllabus-jingjifa-2026.md`](../data/syllabus-jingjifa-2026.md)（7 章/35 考点/116 知识点）做 **diff**：讲义内容挂到已有考点/知识点节点；新出现的节点单列。
4. 产出**待评审清单**（如 markdown/JSON）：哪些是已有节点的正文补充、哪些是疑似新增知识点、哪些命名与考纲不一致需人工裁决。

## Acceptance criteria

- [ ] 3 份讲义各产出「考点→知识点→正文」结构化结果
- [ ] 页眉页脚剔除干净，正文不串章、不跨页拼接错位
- [ ] 与考纲种子的 diff 清单：已有节点匹配项 / 疑似新增项 / 命名不一致项 三类分开列出
- [ ] **不修改** S4 已锁定的知识点表（本切片只读 + 产出清单）
- [ ] 命名映射不做自动 fuzzy 合并（疑似项一律进人工评审）
- [ ] 抽取/diff 的确定性逻辑有测试（小样本 fixture）

## Blocked by

无 — 文字型抽取 + 只读对比考纲种子（与 S13/S14 可并行）。
