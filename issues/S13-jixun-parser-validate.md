## What to build

一个**离线 Python 解析器**：输入「客观题集训」PDF，输出结构化题目，**不写库**。专为集训格式设计——题目在正文、答案+解析统一在文末，按**题号唯一键**配对（这是 S5 错位坑的反面：用唯一键而非位置）。

来源文件（3 份，`media/`）：`周周老师-2026强化班—第二部分 客观题集训{138003,333804,710399}.pdf`。已实测 138003 份：题号 1–92 连续，答案 1–92 连续，题干集与答案集差集为空，26 个考点。

端到端：

1. fitz 抽全文，**剔除页眉页脚**（`第X章…|强化班-周周` / `学会计就到之了课堂`）。
2. **切两段**：以首个 `^\d+[.、]【答案】` 为界，前=题目区、后=答案解析区。
3. 题目区：`^(\d+)[.、]【(单选题|多选题|判断题)】(题干)` 切题；`^[A-D].` 收选项；`^考点\d+[:：](标题)` 作为当前考点上下文附到其后题目。
4. 答案区：`^(\d+)[.、]【答案】(值)` + `【解析】(正文，至下一题号止)`；判断题答案 `√/×`，选择题 `A–D` 组合（统一排序存储）。
5. 按**题号** join 题目与答案。
6. 章节从首页 `第X章 ……` 提取，整份归一章。

输出结构化结果（如 JSON/JSONL 落盘或返回），字段：题号、题型、题干、选项、正确答案、解析、考点、章节、来源。**本切片不触碰 DB**。

## Acceptance criteria

- [ ] 解析器对 3 份集训 PDF 各自产出结构化题目列表
- [ ] 内建校验报告：每份题号是否连续、答案题号是否连续、**题干集与答案集差集是否为空**（任一不满足即标红/报警）
- [ ] 三份各自题号范围独立校验（避免跨文件题号撞车的误判）
- [ ] `--dry` 模式 dump **≥5 条** `题号 ↔ 题干 ↔ 正确答案 ↔ 解析首句` 供人工抽查对齐
- [ ] 多选答案统一排序（`ADB`→`ABD`）；判断题答案规范为 `√/×`
- [ ] 题号切分锚定行首 + `【答案】/【解析】` 标记，解析正文内嵌的 `数字.` 不误切
- [ ] 解析逻辑有确定性测试（mock/小样本 fixture，不依赖真实 API）
