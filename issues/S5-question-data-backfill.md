# S5 — 题库数据补全（解析页配对回填答案/解析）

- **类型**：AFK
- **批次**：第二批
- **状态**：✅ 核心完成；23 道可练、39 道待确认（讲义结构限制）
- **覆盖用户故事**：10–16

## What to build

把被当作垃圾页过滤的「解析页」（含答案+解析）按位置配对、模型校验后，补回对应真题，产出约 60 道完整真题。

- 离线脚本：在每份 PDF 内按「真题在前、解析在后、顺序一致」做**位置配对**（真题[i] ↔ 解析[i]）。
- 用**文本模型校验**每对是否匹配：解析中的 `【答案】X` 与真题现有答案交叉核对、解析场景与题干场景一致。
- 匹配 → 把答案 + 解析**回填**到真题，并把独立解析页记录**移出题库**（不再作为"题"）。
- 不匹配 / 低置信 → 进**人工确认队列**（沿用 `needs_review`）。PDF1 的 27:26 错位靠校验暴露。

> 来自 S2 临时护栏的判定规则（已验证）：解析片段特征 = 题干含 `【答案】`/`【解析】`，或单选/多选但 0 选项。

## Acceptance criteria

- [x] 同一 PDF 内真题与解析按「题号重置切小节 + 节内题号匹配」配对
- [x] 答案交叉校验定置信度（纯函数，未接文本模型做场景校验——PRD 里的模型校验留后续增强）
- [x] 高置信匹配对把答案、解析回填到对应真题
- [x] 高置信解析页记录补全后移出题库（删 36 条）
- [x] 不匹配/低置信进人工确认队列（26 道 `needs_review=1`）
- [ ] 产出约 60 道「题干+选项+答案+解析」齐全的真题（见下方数据事实）
- [x] 配对/回填的确定性逻辑有测试（`tests/test_backfill.py`）
- [x] 真 PDF 重导 + 补全 + 抽查（`scripts/backfill_preview.py`）

## Blocked by

无 — 操作现有题库与解析页数据即可（与 S4 可并行）。

**当前数据状态**：questions 表已被错误补全污染（PDF2 的 35 道按题号匹配可信；PDF1 约 19 道可疑）。**待办：清表重新导入两份 PDF 恢复原始数据 → 跑修复后的补全 → 抽查配对正确**。知识点表（116）不受影响。
