# S14 — 集训题导入题库并可练

- **类型**：AFK
- **批次**：第三批（media 讲义批量入库）
- **状态**：⬜ 待开始
- **来源**：[`PLAN-A-集训题库导入.md`](../PLAN-A-集训题库导入.md)
- **VLM 成本**：0（消费 S13 的结构化结果，无 AI 回填）

## What to build

把 S13 校验通过的集训题**写入题库并直接可练**。集训来源权威（答案+解析现成、题号对齐已验证），因此入库即「可练」，**不进 `needs_review`**——与 S5 的「待确认」路径区分开。

端到端：

1. 读 S13 输出的结构化集训题。
2. 写入现有 `questions` 表：`source=集训-周周-{章}`、考点存为文本字段、状态可练。考点→知识点的归类**交给已有 S6**（本切片只存考点文本，不做分类）。
3. **去重**：与现有题库按题干/来源+题号撞车保护，避免重复导入。
4. 三份集训一次性全导（均文字型、免费）。

**强制吸收 S5 教训（破坏性数据操作）**：

- 导入前**备份 `mistakegenie.db`**。
- 走**可逆**路径：能 flag 不删；不静默覆盖已有题。
- 入库后**逐条抽查**若干题 `题干↔答案↔解析` 确认正确，而非只看「导入 N 道」汇总数字。

## Acceptance criteria

- [ ] 三份集训题入库，`source` 标记正确、考点字段非空、状态为可练（非 needs_review）
- [ ] 导入前自动/手动备份 DB（记录备份路径）
- [ ] 去重生效：重复跑不产生重复题
- [ ] 入库脚本打印汇总（导入 N 道 / 跳过重复 M 道 / 各章分布）
- [ ] 入库后 dump ≥5 道实库样本人工抽查，确认题干·答案·解析对齐正确
- [ ] 在现有刷题流程（今日任务/按章）能抽到这些集训题并正确判分
- [ ] 入库映射逻辑有测试（mock DB / 内存库）
