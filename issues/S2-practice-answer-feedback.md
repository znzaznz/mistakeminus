# S2 — 刷一道题，看到对错 + 正确答案 + 解析

- **类型**：AFK
- **批次**：第一批（最小闭环骨架）
- **状态**：✅ 已完成
- **覆盖用户故事**：14、15（+ 取题）

## What to build

核心刷题交互的最薄端到端版本：用户打开刷题页 → 看到一道题 → 选答案 → 提交 → 立刻看到对错、正确答案和解析。每次作答都落库。

- **后端**：
  - 取题接口：从题库返回若干道题供练习（本切片不做「每日 30 道智能出题」算法，先简单返回一批题，比如按顺序或全部新题）。
  - 作答接口：接收某题的用户答案，判定对错，记录一条作答（题目ID、用户选择、对错、时间），返回对错 + 正确答案 + 解析。
- **前端**：刷题页展示一道题（题干/选项，带图则显示配图），用户选择并提交，即时展示对错、正确答案、解析，可进入下一题。

判题与作答记录逻辑放在后端纯函数层，便于测试。

## Acceptance criteria

- [x] 取题接口返回结构正确（题干/选项/配图/题型），接口级测试覆盖
- [x] 作答接口正确判定单选/多选/判断的对错，返回对错 + 正确答案 + 解析
- [x] 每次作答落库（题目ID、用户选择、对错、作答时间）
- [x] 前端刷题页能展示题目（含配图）、提交答案、即时显示对错/正确答案/解析、进入下一题
- [x] 后端判题纯函数有单测（覆盖单选/多选/判断及边界）
- [x] 取题/作答接口契约有接口级测试
