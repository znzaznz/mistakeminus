# S3 — 答错自动进错题本 + 错题本列表查看

- **类型**：AFK
- **批次**：第一批（最小闭环骨架）
- **状态**：✅ 已完成
- **覆盖用户故事**：16、17（自动收录 + 字段追踪）

## What to build

闭环的最后一环：用户在刷题时答错，这道题自动进入错题本并记录追踪字段；用户能在错题本列表页看到它。

- **后端**：作答判错时（衔接 S2 的作答接口），把该题写入/更新错题本记录。追踪字段至少含：用户错误答案、正确答案、错误次数、答对次数、第一次做错时间、最近一次做题时间、掌握状态（默认「未掌握」）。同一题再次做错则错误次数 +1、更新最近时间，而非重复新增。错题本列表查询接口。
- **前端**：错题本页，列表展示错题（题干、所属考点、错误次数、掌握状态等关键字段）。

错题状态转移（作答 → 错误次数/答对次数/最近时间/掌握状态）放在后端纯函数层，便于测试。本切片只做「自动收录 + 列表查看」，筛选/搜索/收藏/手动标记掌握/截图上传留后续批次。

## Acceptance criteria

- [x] 答错时该题自动进错题本，记录全套追踪字段（用户错误答案、正确答案、错误次数、答对次数、首次做错时间、最近做题时间、掌握状态=未掌握）
- [x] 同一题重复做错 → 错误次数累加、更新最近时间，不重复新增记录
- [x] 错题本列表接口返回正确，接口级测试覆盖
- [x] 前端错题本页能列出错题及关键字段
- [x] 错题状态机（作答→字段转移）有后端纯函数单测，覆盖首次做错/重复做错边界
- [x] 端到端可演示：刷题答错 → 该题出现在错题本

## 验收记录（2026-06-18）

- 后端：`mistakes` 表（一题一行）；状态机 `mistakes.apply_attempt`（纯函数）；作答端点判错后自动 upsert；`GET /mistakes` 列表（join 题目信息，按最近作答倒序）。
- 状态转移：未在本+答对→不收录；未在本+答错→新建(错1/未掌握/记首次时间)；已在本+答错→错次+1、更新最近错答与时间、首次时间不变；已在本+答对→对次+1。掌握状态升级/降级留后续批次。
