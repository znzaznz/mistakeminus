# MistakeGenie 第二阶段 PRD — 知识点体系 · 数据补全 · 智能复习

> 状态：待评审 → 通过后进入 `/to-issues` 拆解
> 前置：第一批最小闭环（S0–S3：导入→刷题→错题本）已完成，后端 41 测试通过。
> 范围：第二批。聚焦「让题库真正可用 + 按官方考纲组织知识 + 智能生成与复习」。

---

## 一、Problem Statement（问题）

第一批闭环跑通后，实际试用暴露出几个让 App「还不能真正拿来备考」的问题：

- **题目残缺**：60 道真题里 18 道没答案、几乎全都没解析——因为答案和解析在单独的「解析幻灯片」上，被当作垃圾页过滤了。现在等于「只有题干」，做了也判不了分、看不到解析。
- **没有知识体系**：题目上的「考点」是 VLM 逐页瞎编的自由文本（123 条里 97 个不同值），不是干净的考纲结构。导致「哪个知识点弱」无从谈起，相似题也没有锚点。
- **不能针对性加练**：想就某个错题/知识点反复练同类题，没有「生成相似题」。
- **复习全凭感觉**：每日任务只是随机抓一组，没有按遗忘曲线安排「临近遗忘才复习」。
- **系统外的错题进不来**：在别处看到的错题想拍照收进来，没有上传识别入口。
- **重点错题无法单独标记**：不能收藏。
- **只能在浏览器用**：还不能下载安装成桌面 App。

## 二、Solution（解决方案）

以**官方考试大纲**为权威骨架，把题库补全、按知识点重新组织，并在其上长出智能生成与智能复习：

- **知识点体系**：引入两层「考点 → 知识点」结构，直接取自 2026 财政部官方考纲（经济法 7 章 / 35 考点 / 116 知识点，每个带官方能力要求）。题目挂到知识点；知识点带「要义总结」，作为相似题生成与薄弱点分析的锚。
- **数据补全**：把被过滤的解析页按「真题在前/解析在后」位置配对 + 模型校验，补回每道真题的答案与解析，产出 60 道完整真题。
- **题目归类**：现有真题自动归类到第一章的 9 个知识点。
- **相似题**：在错题现场按需生成同知识点新题，立即可练。
- **遗忘曲线**：用 SM-2 安排每日复习，质量分自动从对错推（不打扰用户）。
- **截图上传**：复用识题管线，把系统外的错题拍照收进错题本。
- **收藏**：标记并筛选重点错题。
- **可下载**：最后用 Tauri 套壳成桌面安装包。

全程仍本地单用户、免登录、不上云。

## 三、User Stories（用户故事）

### 知识点体系（考点 → 知识点）
1. 作为备考用户，我想让题目按官方考纲的「考点 → 知识点」组织，以便我的复习和分析有权威结构可依。
2. 作为备考用户，我想每个知识点带官方能力要求（掌握/熟悉/了解），以便我知道哪些是重点。
3. 作为备考用户，我想系统先从官方考纲生成一份「考点/知识点」草稿，以便我不用手工录入整棵树。
4. 作为备考用户，我想对这份知识点树做评审/修正后锁定，以便它成为权威基准。
5. 作为备考用户，我想每个知识点有一段「要义总结」（从讲义概括），以便我快速回顾该点，也作为出相似题的依据。
6. 作为备考用户，我想要义草稿由模型生成、我能修改确认，以便兼顾效率与准确。
7. 作为备考用户，我想每道题都归属到某个知识点，以便按知识点刷题、分析、生成。
8. 作为备考用户，我想题目自动归类的结果带置信度、低置信的进人工确认队列，以便我只需复核可疑项。
9. 作为备考用户，我想暂时没题的知识点也存在于体系里，以便以后导入新题能挂上去、也能看出覆盖空白。

### 题库数据补全
10. 作为备考用户，我想每道真题都补回正确答案，以便做完能判分。
11. 作为备考用户，我想每道真题尽量补回解析，以便做错能看懂。
12. 作为备考用户，我想系统把「解析页」自动配对回对应真题，以便我不用手工对。
13. 作为备考用户，我想配对由「位置 + 模型校验」双保险，以便不会因错位把答案配错。
14. 作为备考用户，我想配对不确定的进人工确认队列，以便我复核而不是被静默配错。
15. 作为备考用户，我想补全后那些独立的解析页不再作为「题」出现，以便题库干净。
16. 作为备考用户，我想最终得到约 60 道「题干+选项+答案+解析」齐全的真题，以便正经刷。

### 相似题生成
17. 作为备考用户，我想在某道错题上点「再练一道同知识点的」，以便针对薄弱点加练。
18. 作为备考用户，我想相似题以「该知识点要义 + 原题」为锚生成，以便考点一致、难度思路相近。
19. 作为备考用户，我想生成的相似题自带答案和解析，以便能判分能看懂。
20. 作为备考用户，我想相似题生成后立即可答，以便不被审核流程打断。
21. 作为备考用户，我想相似题被标记「AI 生成」，以便我知道它不是真题。
22. 作为备考用户，我想对质量差的相似题一键报错删除，以便不污染题库。
23. 作为备考用户，我想相似题也归属到原题的知识点，以便统计与复习一致。

### 遗忘曲线每日复习
24. 作为备考用户，我想每天有一份固定的「今日任务」，以便照着做就行。
25. 作为备考用户，我想每日任务按 SM-2 把「临近遗忘」的题优先排进来，以便在忘掉前复习。
26. 作为备考用户，我想质量分由我的对错自动推算，以便我不用每题手动评分。
27. 作为备考用户，我想答对的题复习间隔自动拉长、答错的打回重来，以便复习节奏跟着掌握度走。
28. 作为备考用户，我想每日任务在到期复习题不足时用新题/相似题补足到目标题量，以便每天题量稳定。
29. 作为备考用户，我想能在设置里调整每日题量，以便配合我的时间。
30. 作为备考用户，我想看到今日任务的完成进度，以便知道还剩多少。
31. 作为备考用户，我想出题时参考知识点的官方能力要求（重要程度），以便重点考点优先。

### 截图上传识题
32. 作为备考用户，我想拍照或截图上传一道系统外的错题，以便把它也收进来。
33. 作为备考用户，我想上传的图被视觉模型识别成题干/选项/答案/解析，以便它和其他题一样可用。
34. 作为备考用户，我想识别结果带置信度、我能确认/修改后再入库，以便保证准确。
35. 作为备考用户，我想上传的题被归类到某个知识点，以便纳入统一的分析与复习。
36. 作为备考用户，我想上传的原图被保存且可收藏，以便保留原始材料。
37. 作为备考用户，我想上传识别出的错题进入错题本，以便和自动收录的错题一起复习。

### 错题收藏
38. 作为备考用户，我想收藏某些重点错题，以便单独关注。
39. 作为备考用户，我想按「收藏」筛选错题本，以便快速回到重点。

### 可下载 App
40. 作为备考用户，我想把它下载安装成桌面 App，以便不依赖浏览器、双击就能用。

## 四、Implementation Decisions（实现决策）

### 知识点体系（地基）
- 新增两张表：**`exam_points`（考点）** 与 **`knowledge_points`（知识点）**，`knowledge_points` 外键挂到 `exam_points`，并带 `mastery_requirement`（掌握/熟悉/了解，作"考点重要程度"权重）与 `essence`（要义总结）字段。
- **`questions` 新增 `knowledge_point_id` 外键**；保留原 `chapter/exam_point` 自由文本字段作过渡/参考，不再用于分析。
- **权威来源 = 2026 官方考试大纲**（非讲义、非网络二手）。已用 PyMuPDF 从财政部会计司官方 PDF 解析出经济法全部 **7 章 / 35 考点 / 116 知识点 + 能力要求**，落成种子 `data/syllabus-jingjifa-2026.json`（程序用）与 `.md`（人看）。116 知识点与 116 能力要求精确对齐作为校验。
  - 来源：http://kjs.mof.gov.cn/zhengcefabu/202604/t20260409_3987210.htm
- **种子载入**：独立脚本把该 JSON 灌入 `exam_points/knowledge_points`（幂等）。当前题库仅覆盖第一章 9 个知识点，其余 107 个先建好、暂无题。
- **要义总结**：文本模型从刘琪讲义对应页概括出每个知识点的 `essence` 草稿 → 人工锁定。讲义 + 真题作为对官方结构的交叉验证。
- **题目归类**：现有 60 道真题用文本/视觉模型分类到第一章 9 个知识点之一，带**置信度**；低置信进**人工确认队列**（沿用 S1 的 `needs_review` 模式）。

### 题库数据补全
- 离线脚本：在每份 PDF 内按「真题在前、解析在后、顺序一致」做**位置配对**（真题[i] ↔ 解析[i]），再用**文本模型校验**每对是否匹配（解析中的【答案】与现有答案交叉核对、解析场景与题干场景一致）。
- 匹配 → 把解析页的**答案 + 解析回填**到对应真题，并把独立解析页记录**移出题库**（不再作为"题"）。
- 不匹配/低置信 → 进**人工确认队列**。处理 PDF1 的 27:26 错位即靠此校验发现。
- 来自原型的判垃圾规则（已在 S2 临时护栏验证）：解析片段特征为「题干含 `【答案】`/`【解析】`」或「单选/多选但 0 选项」。

### 相似题生成
- 错题详情页「再练一道同知识点」→ 后端文本模型以 `该知识点.essence + 原题` 为提示生成新题（题干/选项/答案/解析），写入 `questions`：`source=相似题生成`、`knowledge_point_id=原题的`、`year=null`、带「AI 生成」标记。
- 生成即可练，不进确认队列；用户可一键「报错删除」该相似题。

### 遗忘曲线 / 每日任务
- 新增 **SM-2 调度纯函数**：输入（上次 ease、interval、repetition、本次质量分、now）→ 输出（新 ease、新 interval、下次到期日）。**质量分由对错自动映射**（答对→良好档、答错→重来档），不要求用户手动评分。
- 错题/已掌握题的 SM-2 状态随作答更新（衔接现有错题状态机 `apply_attempt`）。
- 新增 **每日任务生成纯函数 + 持久化**：每天生成一次「今日任务」并落库，记录完成进度；组成 = **到期复习题（SM-2 due）优先 → 新题/相似题补足**到目标题量；目标题量可在设置调整；出题时参考知识点 `mastery_requirement` 加权。
- 取题接口从 S2 的「随机一组」升级为「今日任务」。

### 截图上传识题
- 复用 S1 的 VLM 管线（图 → 结构化题）。新增上传接口：保存原图（本地 media，可收藏）→ VLM 识别成草稿（带置信度）→ 人工确认/修改 → 入库并归类知识点 → 作为错题进入错题本。

### 错题收藏
- `mistakes` 表加 `favorite` 标记；错题本列表接口支持按收藏筛选。

### 可下载 App
- 用 **Tauri** 把现有 Web 前端套壳成桌面安装包，后端随应用本地启动。排在本阶段最后，先做完 Web 功能。

### 统一约定
- 所有「识别/分类/配对」类模型调用一律带**置信度 + 人工确认队列**，沿用 S1 既有 `needs_review` 模式。
- 模型调用仍统一封装在后端，走 DashScope（VL + 文本），key 只在后端。

## 五、Testing Decisions（测试决策）

沿用第一阶段已确立的三类测试缝（S1–S3 的既有模式），优先用最高层的缝，不测实现细节：

- **后端纯函数缝**（主力）：
  - **SM-2 调度**：给定上次状态 + 质量分，断言新 ease/interval/到期日与对错→质量分映射正确，覆盖首次、连对升级、答错打回边界。
  - **每日任务生成**：给定题库 + 错题 SM-2 状态 + 目标题量，断言「到期优先 + 补足、去重、题量、能力要求加权」正确，覆盖到期不足/全新用户边界。
  - **错题状态机**（已存在 `apply_attempt`）：扩展收藏/SM-2 字段转移后补测。
  - **解析↔题配对** 与 **题→知识点分类** 的确定性部分（位置配对、置信度阈值分流），**模型调用 mock**。
- **HTTP 接口契约缝**：知识点/考点查询、相似题生成、今日任务、截图上传、收藏筛选接口的请求-响应形状（用 FastAPI TestClient + 临时库注入，沿用 S2/S3 写法）。
- **离线脚本缝**：考纲种子载入幂等、数据补全脚本的确定性产物（配对结果结构、回填字段、解析页移出），VLM/文本模型一律 mock，不打真实 API。
- 真实模型调用（要义抽取、真补全、真分类、真生成）作为**人工验收**单独跑，不进单测。
- 前端以核心交互冒烟为主。

## 六、Out of Scope（本阶段不做）

- 账号系统、多用户、云端同步（延续 V1 约定）。
- 其余章节（公司法/合伙/物权/合同/金融/财政）的**题目**导入——体系已建好占位，但本阶段题库仍聚焦第一章；扩题是后续。
- 多科目（仅经济法；数据模型不写死科目）。
- 导出错题本（PDF/Word/Excel）。
- 统计图表/趋势可视化（薄弱点分析以知识点维度数据为主，富图表后续）。
- 手机原生 App / 微信小程序（桌面 Tauri 之外不做）。
- SM-2 的手动质量分评分 UI（本阶段一律自动推算）。

## 七、Further Notes（补充）

- 官方考纲提取已完成并落库为种子文件，是本阶段「知识点体系」决策的实证依据；能力要求列一并解决了 V1 PRD 里「考点重要程度」这一直悬空的出题输入。
- 命名说明：刘琪强化班的「7 考点」是把官方知识点拆得更细的讲法，可干净映射到官方「考点→知识点」之上；以官方结构为准。
- 建议拆分顺序（地基优先）：知识点体系建库 → 数据补全 → 题目归类 → 薄弱点分析（知识点维度）→ 相似题 → 每日任务(SM-2) → 截图上传 → 收藏 → App 套壳。
- 第一阶段发现的两个数据质量临时护栏（S2 取题过滤解析片段）在数据补全完成后可移除或收紧。
