# 计划 A · 客观题集训题库导入

> 优先级：🔥 **最高**（价值最高、零回填风险，应最先做）
> 来源：`media/周周老师-2026强化班—第二部分 客观题集训{138003,333804,710399}.pdf`（3 份，文字型）
> **VLM 成本：0**（文字型，全程 fitz 抽文字，不调任何视觉/AI 回填）

## 1. 为什么先做这个

这 3 份是**带答案带解析的结构化题库**，是目前最干净的题源：

- 题目按「考点」分组，每题带 `【单选题】/【多选题】/【判断题】` 标签
- **正文出题（题号 1→N 连续），文末统一附 `N.【答案】… 【解析】…`**
- 已实测（138003 份）：题号 **1–92 完全连续**，答案 **1–92 完全连续**，题干集与答案集**差集为空**，题号 1:1 对齐
- 题型分布样本：单选 30 / 多选 50 / 判断 12，26 个考点

**关键意义**：答案和解析现成写在 PDF 里，**完全不需要 AI 回填答案**，因此**根本不会重演 S5 的按位置配错位 + 烧钱重导**。这是 S5 的反面教材——用唯一键（题号）配对、两侧可验证对齐。

## 2. 目标

把 3 份集训 PDF 解析成结构化题目（题干 / 选项 / 题型 / 正确答案 / 解析 / 考点 / 章节），导入现有题库，状态直接为「可练」（无需待确认）。

## 3. 与现有 S1 的差异

现有 S1 导入器假设「题干在正文、解析靠 AI 补」。集训格式不同，需要**新增一个解析分支 / 专用 parser**：

| 维度 | S1 现状 | 集训需要 |
|---|---|---|
| 答案来源 | AI 推断/回填 | **PDF 文末已有，按题号锚定** |
| 解析来源 | AI 生成 | **PDF 文末已有** |
| 分组 | 章节 | 章节 + **考点（考点N：标题）** |
| 置信度 | 需人工确认 | 直接可练（来源权威） |

## 4. 解析器设计

1. **切两段**：以第一个 `\d+.【答案】` 出现处为界，前半=题目区，后半=答案解析区。
2. **题目区**：正则 `^(\d+)[.、]【(单选题|多选题|判断题)】(题干)` 切题；选项按 `^[A-D].` 收集；`^考点\d+[:：](.*)` 作为当前考点上下文，附到其后题目。
3. **答案区**：正则 `^(\d+)[.、]【答案】(值)` + `【解析】(正文，至下一题号止)`。判断题答案为 `√/×`，选择题为 `A–D` 组合。
4. **按题号 join**：题目.题号 = 答案.题号，**唯一键配对**（非位置）。
5. **章节**：从首页 `第X章 ……` 提取，整份归一章。

## 5. 数据模型影响

- 复用现有题目表；确认有 `考点/keypoint` 字段，没有则加（与 S4 知识点体系挂钩，见计划 B 与 S6）。
- 标注来源 `source=集训-周周-{chapter}`，便于回溯与去重。

## 6. 验证（吸取 S5 教训，强制）

- [ ] 解析器先 `--dry` 输出：题目数 / 答案数 / 题号是否连续 / **差集是否为空**（任一不满足即报警停止）
- [ ] **逐条抽查 ≥5 题**：dump `题号 ↔ 题干 ↔ 正确答案 ↔ 解析首句`，人工核对对齐与正确性（只看汇总数字不够）
- [ ] 三份分别校验题号范围（各自 1→N），避免跨文件题号撞车
- [ ] 入库走可逆路径：能 flag 不删；导入前备份 `mistakegenie.db`

## 7. 风险

- 文末解析正文里可能内嵌 `\d+.` 干扰切分 → 题号切分锚定行首 + `【答案】/【解析】` 标记，别只靠数字。
- 多选答案顺序（`ABD` vs `ADB`）→ 统一排序后存储。
- 选项跨行 / 题干含括号占位 `（ ）` → parser 容错。

## 8. 工作量

约 1 个 issue（集训专用 parser + dry 校验 + 三份导入），半天量级。建议拆为：A1 写 parser+校验、A2 三份实导入并抽查。
