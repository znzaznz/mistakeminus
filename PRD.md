# MistakeGenie 刷题错题 App — PRD（第一版）

> 状态：待评审 → 通过后进入 `/to-issues` 拆解
> 范围：第一版（V1），目标是「能跑起来、能自己用、能验证核心闭环」

---

## 一、Problem Statement（问题）

我（备考用户，当前科目：中级会计《经济法》）刷题时遇到几个痛点：

- 做错的题散落各处，没有一个地方自动沉淀、归类、追踪。
- 不知道自己哪些知识点弱，复习全靠感觉。
- 想针对一类题反复练，但没有"同考点相似题"可练。
- 复习节奏没人安排，今天该复习哪些早就忘了的题，全凭记忆。
- 手上有讲义 PDF（刘琪老师强化班，含真题+答案+解析），但题被困在 PDF 里，用不起来；偶尔在别处看到错题想随手拍下来收进来也没地方放。

## 二、Solution（解决方案）

一个**本地单用户的 Web 刷题工具**：

- 把讲义 PDF 自动抽成结构化题库（题干/选项/答案/解析/考点/年份/配图）。
- 每天自动生成约 30 道刷题任务（新题 + 历史错题 + 相似题 + 遗忘曲线复习题）。
- 做错的题自动进错题本，记录全套追踪字段。
- 支持拍照/截图上传错题，由视觉大模型识别成结构化题入库。
- 用大模型分析薄弱知识点、生成相似题、生成/补全解析、总结错因。
- 按遗忘曲线安排每日复习，追踪正确率、掌握度、学习趋势。

全程本地运行、免注册、不上云。

## 三、User Stories（用户故事）

### 题库与资料导入
1. 作为备考用户，我想把一份讲义 PDF 导入系统，以便把里面的真题变成可刷的题库。
2. 作为备考用户，我想让系统自动识别 PDF 里每道题的题干、选项、正确答案、解析、所属考点和年份，以便我不用手动录入。
3. 作为备考用户，我想让 PDF 里题目带的图示/表格也被一并保存并显示，以便带图的题不丢信息。
4. 作为备考用户，我想在导入后看到「成功识别 N 道题、M 道需人工确认」的结果，以便我知道导入质量。
5. 作为备考用户，我想对识别可疑的题做人工修正，以便题库准确。
6. 作为备考用户，我想之后还能再导入别的 PDF，以便逐步扩充题库。

### 日常刷题
7. 作为备考用户，我想每天打开就看到当天的刷题任务（默认约 30 道），以便不用自己决定刷什么。
8. 作为备考用户，我想每日任务由「新题 + 历史错题 + 相似题 + 遗忘曲线复习题」按比例组成，以便兼顾学新和复习。
9. 作为备考用户，我想在设置里调整每日题量，以便按自己的时间安排。
10. 作为备考用户，我想按章节刷题，以便专项突破。
11. 作为备考用户，我想按考点刷题，以便针对性练习。
12. 作为备考用户，我想按错题刷题，以便重做做错的题。
13. 作为备考用户，我想按薄弱知识点刷题，以便补短板。
14. 作为备考用户，我想答完一题立刻看到对错、正确答案和解析，以便即时学习。
15. 作为备考用户，我想我的每次作答都被记录（选了什么、对错、时间），以便系统据此分析。

### 错题本
16. 作为备考用户，我想做错的题自动进错题本，以便不用手动收集。
17. 作为备考用户，我想错题本记录题目内容、我的答案、正确答案、解析、章节、考点、错误次数、复习次数、最近做题时间、掌握状态，以便全面追踪。
18. 作为备考用户，我想收藏某些重点错题，以便单独关注。
19. 作为备考用户，我想按章节/考点/掌握状态筛选错题，以便有条理地复习。
20. 作为备考用户，我想搜索错题，以便快速定位。
21. 作为备考用户，我想手动标记某题"已掌握/未掌握"，以便修正系统判断。
22. 作为备考用户，我想拍照或截图上传一道错题，以便把系统外的题也收进来。
23. 作为备考用户，我想上传的截图被保存且可收藏，以便保留原图。
24. 作为备考用户，我想上传的截图被视觉模型识别成题干/选项/答案/解析并归类考点，以便它和其他错题一样可复习。
25. 作为备考用户，我想对识别结果做确认/修改后再入库，以便保证准确。

### AI 分析与生成
26. 作为备考用户，我想看到系统分析出的薄弱知识点（高频错考点/章节、易错题型、重复做错的题、正确率下降的考点、长时间未复习的考点），以便知道该补哪。
27. 作为备考用户，我想针对一道错题生成相似题（同考点、难度相近、题型相似、问法有变化），以便强化训练。
28. 作为备考用户，我想对缺解析的题由系统生成解析，以便看懂。
29. 作为备考用户，我想系统总结我这道题的错误原因，以便对症。
30. 作为备考用户，我想看到系统推荐的复习重点，以便集中精力。

### 复习计划（遗忘曲线）
31. 作为备考用户，我想系统按遗忘曲线把"临近遗忘"的题排进每日任务，以便在忘掉前复习。
32. 作为备考用户，我想复习排期结合首次做错时间、最近复习时间、错误次数、正确次数、当前掌握状态、考点重要程度，以便排得合理。
33. 作为备考用户，我想某错题连续答对后出现频率自动降低，以便不浪费时间在已掌握的题上。
34. 作为备考用户，我想某考点连续做错时它的题目占比自动提高，以便针对薄弱处加练。
35. 作为备考用户，我想看到今天还有哪些复习任务没做，以便完成当日计划。

### 数据统计
36. 作为备考用户，我想看到总刷题数、正确率、错题数量、重复错题数量，以便了解整体情况。
37. 作为备考用户，我想看到各知识点掌握度，以便定位强弱。
38. 作为备考用户，我想看到每日学习趋势，以便看到进步。
39. 作为备考用户，我想看到章节掌握情况和复习完成情况，以便把握进度。

## 四、Implementation Decisions（实现决策）

### 架构
- **形态**：本地单用户 Web 应用，免登录、无账号、不上云。
- **前端**：React + Vite（TypeScript）。
- **本地后端**：Python + FastAPI。后端持有千问 API key（key 绝不进浏览器），对外提供刷题/错题/分析/AI 接口；每日出题、遗忘曲线、掌握度等核心逻辑在后端。
- **数据库**：SQLite（错题字段多、需计算遗忘曲线，关系型存储优于浏览器本地存储）。
- **图片存储**：本地文件夹，数据库存相对路径。
- **题库提取**：独立 Python 脚本（离线运行），PDF → 题库写入 SQLite。

### AI / 模型层（统一一个 provider）
- **Provider**：通义千问 / DashScope（阿里云百炼），OpenAI 兼容接口。
- **视觉模型**（`QWEN_VL_MODEL`，默认 `qwen-vl-max`）：PDF 逐页识题、拍照/截图识题。
- **文本模型**（`QWEN_TEXT_MODEL`，默认 `qwen-plus`）：生成相似题、生成/补全解析、总结错因、薄弱点分析。
- **配置**：全部走 `.env`（`DASHSCOPE_API_KEY` / `DASHSCOPE_BASE_URL` / `QWEN_VL_MODEL` / `QWEN_TEXT_MODEL`）。`.env` 已在 `.gitignore`。
- AI 调用统一封装在后端一个模块，便于以后换 provider（接口不变，换 SDK 即可）。

### PDF 题库提取（已验证）
- **主路径**：用 PyMuPDF 把每页**整页渲染成图**（`page.get_pixmap`），整页图喂给 VLM，直接返回结构化 JSON（题干/选项/答案/解析/考点/年份/是否含图）。对版式鲁棒，图文一起理解，34 页一次性成本 < 1 元。
- **配图抽取**：对带图的题，用嵌入图的**坐标框**（`page.get_images` + `page.get_image_rects`）定位 → 与题干文字 y 坐标比对确定归属 → 用**裁剪渲染**（`page.get_pixmap(clip=bbox)`）按看到的样子重渲染出该图，存入该题 `图片` 字段。
- **明确禁用**：不要用 `Pixmap(doc, xref)` 直接抠原始嵌入图对象——实测会因透明遮罩/颜色空间问题得到全黑图。要图一律「按坐标裁剪渲染」。
- VLM 识别结果带**置信度/需确认**标记，低置信度的题进人工确认队列。

### 数据模型（错题 / 题目核心字段）
每道题/错题至少记录：题目 ID、题目截图（可选）、题目文本、选项、所属章节、所属考点、题型（单选/多选/判断）、难度、年份、正确答案、解析、配图（可选，列表）、来源（PDF导入/截图上传/相似题生成）。
错题额外记录：用户错误答案、错误原因、第一次做错时间、最近一次复习时间、错误次数、答对次数、收藏状态、掌握状态。

### 每日出题逻辑
- 默认每日约 30 道，比例初稿：10 新题 + 10 历史错题 + 5 相似题 + 5 遗忘曲线复习题；题量可在设置调整，比例按题量等比缩放。
- 动态调整：某考点连续做错 → 提高该考点占比；某题型正确率高 → 降低复习频率；某错题多次答对 → 逐步降低出现频率。
- 遗忘曲线排期输入：首次做错时间、最近复习时间、错误次数、正确次数、当前掌握状态、考点重要程度。

### 掌握状态与薄弱点
- 错题掌握状态随作答更新（连续答对升级、再次做错降级），用户可手动覆盖。
- 薄弱点分析维度：高频错考点、高频错章节、易错题型、重复做错的题、正确率下降的考点、长时间未复习的考点。

## 五、Testing Decisions（测试决策）

- **好测试的标准**：只测外部行为/输出契约，不测实现细节。优先在最高层的缝（后端纯函数与 HTTP 接口）打测试。
- **每日出题逻辑**（后端纯函数）：给定一组题目+错题+作答历史，断言生成的 30 道题在数量、比例、去重、动态调整规则上正确。重点测边界（题不够、全是新用户、某考点连错）。
- **遗忘曲线/掌握度计算**（后端纯函数）：给定时间/错对次数/状态，断言排期与掌握状态转移正确。
- **错题状态机**（后端纯函数）：作答 → 错误次数/答对次数/掌握状态/最近时间 的转移正确。
- **PDF 提取脚本的确定部分**：整页渲染产物存在且尺寸正确；嵌入图坐标框能取到；裁剪渲染产物非空且非全黑；VLM 返回的 JSON 通过 schema 校验。**VLM 调用本身 mock 掉**（不确定，不在单测里打真实 API）。
- **HTTP 接口契约**：刷题/错题/统计接口的请求-响应形状用接口级测试覆盖。
- 前端以核心交互冒烟为主，不追求高覆盖。

## 六、Out of Scope（V1 不做）

- 账号系统、注册登录、多用户、云端同步。
- 手机原生 App / 微信小程序 / 打包成可安装 App（V1 先 Web，后续再套壳）。
- 爬取网络题库（合规与稳定性风险，明确放弃）。
- 导出错题本为 PDF/Word/Excel。
- 传统 OCR（已由 VLM 取代）。
- PDF 自动抠图的精确文字/图分离（改为整页喂 VLM + 按坐标裁剪渲染）。
- 多科目支持（V1 聚焦中级会计《经济法》，但数据模型不写死科目）。

## 七、Further Notes（补充）

- 题库初始来源：`刘琪老师-2026强化班—第一章 总论 第一考点-第五考点280619.pdf`、`刘琪老师-2026强化班—第一章 总论 第六考点-第七考点569072.pdf`，共约 60 道真题（单选/多选，2020–2025），均带答案+解析，天然按"第一章 → 7 个考点"组织。
- 这两个 PDF 实为「每页一张幻灯片大图 + 一层文字」结构；文字层可干净抽取，图层用裁剪渲染获取。
- 抠图三方式实测结论：整页渲染 ✅ / 直接抠嵌入图对象 ❌(全黑) / 按坐标裁剪渲染 ✅。`_demo/` 下有验证产物。
- 成本极低：识一张题图约 1~2 分钱，生成一道相似题约几分钱，个人单用户日成本可忽略。
